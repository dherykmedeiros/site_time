# Roadmap 004 — Guia de Navegacao

**Objetivo**: servir como ponto unico de entrada para os artefatos do roadmap, evitando que o time precise redescobrir qual documento usar em cada momento.

---

## Comeco Rapido

### Quero entender a visao geral

- `spec.md`
- `plan.md`
- `priority-scorecard.md`

### Quero saber a ordem real de execucao

- `execution-board.md`
- `tasks.md`
- `operating-rhythm.md`
- `weekly-status-template.md`

### Quero iniciar a Onda 1

- `implementation-readiness-review.md`
- `wave-1-contract-freeze.md`
- `wave-1-engineering-handoff.md`
- `wave-1-sprint-plan.md`
- `wave-1-acceptance-matrix.md`

### Quero abrir uma PR especifica

1. ler o mini-spec da feature;
2. ler o handoff da onda;
3. abrir o checklist da PR;
4. conferir `decision-log.md` e `mvp-cutlines.md`.

### Quero decidir se a proxima onda pode abrir

- `execution-board.md`
- `metrics-and-rollout.md`
- `risk-register.md`
- `implementation-readiness-review.md`
- `decision-log.md`

## Regras de Uso

- use `execution-board.md` para saber ordem;
- use `operating-rhythm.md` para saber como rodar a semana;
- use `decision-log.md` para evitar rediscutir o que ja foi travado;
- use o checklist tecnico da PR como documento principal de implementacao;
- use `metrics-and-rollout.md` antes de decidir expandir, monetizar ou abrir nova onda.
